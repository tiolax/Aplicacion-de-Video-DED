//////ARQUITECTURA//////
----Datos generales---
La app está en un modelo MCV(Modelo-Vista-Controlador)

La base de datos es SQLite (porque no se esperan muchas entradas y es mas facil manejarla)

Los archivos JS estan separados en dos las "Fetchs" y la "logica", la Fecth no es mas que una abreviacion para
pedir informacion a las rutas. (Agunos js no tienen esa separacion)

El flujo es: Cliente -> Logica -> Fecth -> Control -> Modelo -> BD (y se regresa)

---Modelos-----
Los modelos son funciones que manipulan directamente la base de datos, como Crear, Editar,Eliminar,etc.
Prisma sustituye el uso de Querys con funciones prisma para facilitar su uso.

En este nivel solo harás consultas, (Es posible que tenga una que otra validacion extra por ahí, puedes modificarla
sí te sientes determinado).

---Controladores---
Los controladores son el paso intermedio entre el cliente y los modelos, Aquí es donde harás todo tipo de validaciones y middlewares

Asegurate de enviar correctamente los datos al modelo.  
Cada modelo tiene su propio controlador, pero si llegaras a necesitar de dos o mas modelos, he creado una "Capa de Servicio"

En la capa de servicio donde puedes combinar modelos, miralo como un controlador que usa varios modelos.

---Fetch----
Esto es del lado del cliente, En este nivel hacemos las validaciones del cliente (campos vacios, errores alfanumericos,etc..)
Idealmente cada .js tiene su propia coleccion de Fetchs, pero durante el desarrollo me di cuenta que mucho codigo se repetía,
así que verás un par de js que reciclan fetchs. Sí quieres organizarlo y agruparlo puedes hacerlo.


----Sesiones----
Tenemos una tabla de sesiones en la BD, aqui se van creando las sesiones activas. Cuando un usuario se desconeta
la fila correspondiente se borra(recuerda que usamos SQlite y no queremos hacer multiples entradas)

Al momento del logeo, se guarda el id de sesion en LocalStorage, en cada cambio de pagina, la navbar hacer una validacion para verificar
que la sesion siga activa.(Puedes usar Prisma Studio para monitorear la base de datos)

Cuando cierras sesion, la entrada se borra y limpiamos el LocalStorage.

----Css----
Al principio de la creacion todo era un solo archivo llamado "style.css" pero llegando al final sentí la necesidad
de organizarlo, así que comencé a separarlo, pero no lo terminé. Todo es funcional, pero puedes separarlo para 
que se organice mejor

---html---
Realmente aqui no hay mucho que moverle, si quieres meter mano, quita los comentarios verdes y organiza a tu gusto
